Microbiome Analysis Using QIIME

Data used is the “classic” QIIME2 tutorial data from the Earth Microbiome Project. Data is collected from two individuals/subjects. Samples where collected from 4 body sites.

Pipeline

Activate the installed Qiime environment

#| eval: false

# My QIIME env is named as qimme due to typing error during installation process
conda activate qimme2-2026.4

# create working directory
mkdir q2-workshop
cd q2-workshop

Retrieve and store data

Retrieved the metadata and the zip file containing single end sequences, these are saved to the “emp-single-end-sequences” directory. The zipped file is then unzipped.

#| eval: false

wget -O 'sample-metadata.tsv' \
  'https://moving-pictures-tutorial.readthedocs.io/en/2026.4/data/moving-pictures/sample-metadata.tsv'

wget -O 'emp-single-end-sequences.zip' \
  'https://moving-pictures-tutorial.readthedocs.io/en/2026.4/data/moving-pictures/emp-single-end-sequences.zip'
 
unzip -d emp-single-end-sequences emp-single-end-sequences.zip

Import data

Emphasize the “type” of sequences being imported whether illumina or single end sequences. For this tutorial, the sequences are single end/forward reads.

#| eval: false

qiime tools import \
 --type EMPSingleEndSequences \
 --input-path emp-single-end-sequences \
 --output-path emp-single-end-sequences.qza

Demultiplexing

This step separate sequencing reads and assigns them back to the correct samples based on their unique barcode (index) sequences. Expected output file is demux.qza

#| eval: false

qiime demux emp-single \
 --i-seqs emp-single-end-sequences.qza \
 --m-barcodes-file sample-metadata.tsv \
 --m-barcodes-column barcode-sequence \
 --o-per-sample-sequences demux.qza \
 --o-error-correction-details demux-details.qza

Qc step

Visualize the demux.qzv to assess the quality of the reads

#| eval: false

qiime demux summarize \
 --i-data demux.qza \
 --o-visualization demux.qzv

Denoise, depreplicate & additional QC

Denoising is done to correct sequencing errors. For the tutorial, denoising is done with DADA2. Truncation done at 120, beyond this reads showed poor quality

#| eval: false

# Run DADA2 & truncate the sequences from position 120 onward 
# This is due to the quality drop-off observed in demux.qzv 
qiime dada2 denoise-single \
 --i-demultiplexed-seqs demux.qza \
 --p-trim-left 0 \
 --p-trunc-len 120 \
 --o-representative-sequences rep-seqs.qza \
 --o-table table.qza \
 --o-denoising-stats stats.qza \
 --o-base-transition-stats base-transition-stats.qza

Generate feature table and representative sequences

Upload the stats.qzv, rep-seqs.qzv, feature table(table.qzv) into “view.qiime2.org” to view

#| eval: false

# To get stats.qzv
qiime metadata tabulate \
 --m-input-file stats.qza \
 --o-visualization stats.qzv 

# To get table.qzv
# this differs from the command shared in the class presentation due to difference in the qimme2 versions
qiime feature-table summarize \
  --i-table table.qza \
  --m-metadata-file sample-metadata.tsv \
  --o-feature-frequencies feature-frequencies.qza \
  --o-sample-frequencies sample-frequencies.qza \
  --o-summary table.qzv
  
#To get the rep-seqs.qzv
qiime feature-table tabulate-seqs \
 --i-data rep-seqs.qza \
 --o-visualization rep-seqs.qzv

Taxonomy assignment

Download a classifier, assign taxonomy to representative sequences, & visualize taxonomy The taxonomy.qzv object is not required, but may be interesting to view – it matches feature IDs of the rep-seqs to their assigned taxon

#| eval: false

wget \
 -O "16S-classifier.qza" \
 "https://moving-pictures-tutorial.readthedocs.io/en/2026.4/data/moving-pictures/suboptimal-16S-rRNA-classifier.qza"

# Assign taxonomy 
qiime feature-classifier classify-sklearn \
 --i-classifier 16S-classifier.qza \
 --i-reads rep-seqs.qza \
 --o-classification taxonomy.qza

# Output taxonomy.qzv for visualization
qiime metadata tabulate \
 --m-input-file taxonomy.qza \
 --o-visualization taxonomy.qzv
 
# upload the taxonomy.qzv in view.qiime2.org

Taxa bar plots

Use the table.qza to generate a taxa bar plot

#| eval: false

qiime taxa barplot \
 --i-table table.qza \
 --i-taxonomy taxonomy.qza \
 --m-metadata-file sample-metadata.tsv \
 --o-visualization taxa-bar-plot.qzv

Phylogenetic tree construction

#| eval: false

#Create a phylogenetic tree using the mafft algorithm 

qiime phylogeny align-to-tree-mafft-fasttree \
 --i-sequences rep-seqs.qza \
 --o-alignment aligned-rep-seqs.qza \
 --o-masked-alignment masked-aligned-rep-seqs.qza \
 --o-tree unrooted-tree.qza \
 --o-rooted-tree rooted-tree.qza 

# generates a rooted-tree.qza that is put in a phylogenetic analysis tool

Alpha rarefaction

Normalization step. This is random sub setting of reads from each sample, to have the same number of reads per sample. In case the sequencing depth is good, this step can be skipped.

#| eval: false

#The value of --p-max-depth should be inferred from the read depth of your own sample set 
#The more reads you have, the higher you should set this test depth to get an accurate plot 

qiime diversity alpha-rarefaction \
 --i-table table.qza \
 --i-phylogeny rooted-tree.qza \
 --p-max-depth 4000 \
 --m-metadata-file sample-metadata.tsv \
 --o-visualization alpha-rarefaction.qzv

Core diversity metrics

Visualize the object obtained after running alpha rarefaction to determine the sampling depth. Sampling depth is used as 1103 in this tutorial because it is the minimum value of reads from which plateauing is observed. That is after this point, the observed features remain constant with increase in number of reads. Consider point of plateauing, number of samples retained and numer of features retained.

#| eval: false

#The sampling-depth here is something you can choose based off your rarefaction and table visualisations 

qiime diversity core-metrics-phylogenetic \
 --i-phylogeny rooted-tree.qza \
 --i-table table.qza \
 --p-sampling-depth 1103 \
 --m-metadata-file sample-metadata.tsv \
 --output-dir core-metrics-results

Alpha diversity

Diversity vectors are obtained for visualization. For this tutorial shannon and faith-pd were selected.

#| eval: false

#Use the diversity vectors to make visualisations for alpha diversity metrics of choice 
qiime diversity alpha-group-significance \
 --i-alpha-diversity core-metrics-results/shannon_vector.qza \
 --m-metadata-file sample-metadata.tsv \
 --o-visualization core-metrics-results/shannon-group-significance.qzv 

qiime diversity alpha-group-significance \
 --i-alpha-diversity core-metrics-results/faith_pd_vector.qza \
 --m-metadata-file sample-metadata.tsv \
 --o-visualization core-metrics-results/faith-pd-group-significance.qzv

Beta diversity

Compare microbial communities based on body sites or study subjects. Used bray_curtis_metric

#| eval: false

#Body-site and subject both appear to influence beta-diversity – test the statistical significance
qiime diversity beta-group-significance \
 --i-distance-matrix core-metrics-results/bray_curtis_distance_matrix.qza \
 --m-metadata-file sample-metadata.tsv \
 --m-metadata-column body-site \
 --o-visualization core-metrics-results/bray_curtis-body-site-significance.qzv \
 --p-pairwise 

qiime diversity beta-group-significance \
 --i-distance-matrix core-metrics-results/bray_curtis_distance_matrix.qza \
 --m-metadata-file sample-metadata.tsv \
 --m-metadata-column subject \
 --o-visualization core-metrics-results/bray_curtis-subject-group-significance.qzv \
 --p-pairwise 

#We can also generate our own Emperor plots with custom axes 
#These below are for time-series data and show us how samples changed over time 
qiime emperor plot \
 --i-pcoa core-metrics-results/bray_curtis_pcoa_results.qza \
 --m-metadata-file sample-metadata.tsv \
 --p-custom-axes days-since-experiment-start \
 --o-visualization core-metrics-results/bray-curtis-emperor-days-since-experiment-start.qzv

Differential Abundance

Used ancombc plugin First flitered out samples from one body site. Looked at differences based on subjects

#| eval: false

# Filtering the feature table by the column “body-site” where the value is “gut”

qiime feature-table filter-samples \
 --i-table table.qza \
 --m-metadata-file sample-metadata.tsv \
 --p-where "[body-site]='gut'" \
 --o-filtered-table gut-table.qza
 
#Create the differentials & visualise the enriched/depleted taxa between subject 1 & 2; the p-value for significance can be adjusted
qiime composition ancombc \
 --i-table gut-table.qza \
 --m-metadata-file sample-metadata.tsv \
 --p-formula 'subject' \
 --o-differentials ancombc-subject.qza 

qiime composition da-barplot \
 --i-data ancombc-subject.qza \
 --p-significance-threshold 0.001 \
 --o-visualization da-barplot-subject.qzv  
 
 # Compared the subjects based on level 6(Genus)
 qiime taxa collapse \
 --i-table gut-table.qza \
 --i-taxonomy taxonomy.qza \
 --p-level 6 \
 --o-collapsed-table gut-table-l6.qza 

 qiime composition ancombc \
 --i-table gut-table-l6.qza \
 --m-metadata-file sample-metadata.tsv \
 --p-formula 'subject' \
 --o-differentials l6-ancombc-subject.qza 

 qiime composition da-barplot \
 --i-data l6-ancombc-subject.qza \
 --p-significance-threshold 0.001 \
 --p-level-delimiter ';' \
 --o-visualization l6-da-barplot-subject.qzv

Data Exporting

QIIME to R

Data can be exported from QIIME to R. On exporting, folders are created containing files, but file formats change to those compartible with R.

#| eval: false
#Exporting representative features in a folder, as a fasta file
qiime tools export \
 --input-path rep-seqs.qza \
 --output-path rep-seqs 

#Exporting a feature table to BIOM format
qiime tools export \
 --input-path table.qza \
 --output-path table 

#Exporting taxonomy information as a tsv file
qiime tools export \
 --input-path taxonomy.qza \
 --output-path taxonomy 

# First you will need to edit your taxonomy file for biom to recognize it: 
# Edit the taxonomy.tsv exported from the previous command as follows 

# Can use the awk command
awk 'BEGIN{FS=OFS="\t"} NR==1 {gsub(/^Feature ID$/,"#OTUID",$1); gsub(/^Taxon$/,"taxonomy",$2); gsub(/^Confidence$/,"confidence",$3)} {print}' ./taxonomy/taxonomy.tsv > ./taxonomy/taxonomy_tmp.tsv && mv ./taxonomy/taxonomy_tmp.tsv ./taxonomy/taxonomy.tsv

# Then add taxonomy to biom 
biom add-metadata \
 -i table/feature-table.biom \
 -o taxonomy/biom-with-taxonomy.biom \
 --observation-metadata-fp taxonomy/taxonomy.tsv \
 --sc-separated taxonomy
